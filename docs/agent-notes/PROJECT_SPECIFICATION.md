# MakeupLogger 仕様書

この文書は、現行実装を読み取って整理した「実装ベースの仕様書」です。企画上の理想仕様ではなく、コードから確認できた現在の動作と構成を基準にまとめています。

関連メモ:

- `docs/agent-notes/DATA_MODEL.md`

## 1. プロジェクト概要

`MakeupLogger` は、メイク手順や使用カラーを画像ベースで記録する iOS アプリです。

主な目的:

- 顔全体の写真をメイクログとして保存する
- 顔の各パーツごとに個別画像を登録する
- パーツ画像上にアノテーションを置いてメイク位置を記録する
- カラーパレット画像上に色注釈を作成する
- 顔パーツ側の注釈から、使用したカラーパレットと色注釈を紐づける

## 2. 対象環境

- プラットフォーム: iOS
- UI フレームワーク: UIKit
- 言語: Swift 5
- 永続化: Realm
- 依存管理: Carthage
- Realm バージョン: `v10.20.2`
- アプリ本体の Deployment Target: `iOS 15.0`
- テストターゲットの Deployment Target: `iOS 14.2`
- Bundle Identifier: `com.saitoh.k.makeuplogger`

## 3. 権限・外部連携

### 3.1 権限

以下の端末権限を使用します。

- カメラ
  - 用途: メイクした顔やパーツの画像を撮影する
- フォトライブラリ
  - 用途: 既存画像を選択する

### 3.2 外部リンク

- 一覧画面の情報ボタンから SafariViewController で Notion ページを開く

## 4. アプリ起動と構成

### 4.1 起動フロー

- `SceneDelegate` で `UIWindow` を生成する
- ルート画面に `MakeupLogListViewController` を設定する
- `UINavigationController` 配下で画面遷移する

### 4.2 データソース切り替え

- `DEBUG` ビルド時:
  - `MakeupLogRepositoryInMemory`
  - `ColorPalletRepositoryInMemory`
- それ以外:
  - `MakeupLogRealmRepository`
  - `ColorPalletRealmRepository`

開発時はサンプルデータを使った画面確認がしやすい構成です。

## 6. 画面仕様

## 6.1 トップ一覧画面

画面:

- `MakeupLogListViewController`

表示内容:

- セクション 1: メイクアップログ一覧
- セクション 2: カラーパレット一覧

各行の表示:

- メイクログ:
  - サムネイル画像
  - タイトル
- カラーパレット:
  - サムネイル画像
  - タイトル

操作:

- 行タップで詳細画面へ遷移
- 右上の `+` ボタンで追加種別を選択
  - メイクを追加
  - カラーを追加
- 左上の情報ボタンで外部説明ページを開く
- スワイプ削除で対象を削除

一覧更新:

- 画面表示時に再読込する
- `didLogUpdate`
- `didColorPalletUpdate`
  の通知を受けると再読込する

## 6.2 メイクログ新規作成画面

画面:

- `AddNewMakeupLogViewController`

入力項目:

- タイトル
- 説明
- 画像

画像取得方法:

- カメラ撮影
- フォトライブラリ選択

完了条件:

- タイトルが入力済みであること
- 画像が設定済みであること

完了時動作:

- リポジトリへ保存
- 一覧画面へ戻る

エラー表示:

- タイトル未入力時はタイトル欄背景色を変更
- 画像未設定時は画像追加ボタン背景色を変更

## 6.3 カラーパレット新規作成画面

画面:

- `AddNewColorPalletViewController`

入力項目:

- タイトル
- 画像

画像取得方法:

- カメラ撮影
- フォトライブラリ選択

完了条件:

- タイトルが入力済みであること

補足:

- 画像は任意

完了時動作:

- リポジトリへ保存
- 一覧画面へ戻る

## 6.4 メイクログ詳細画面

画面:

- `MakeupLogViewController`

表示構成:

- 上部に画像表示領域
- 画像下にセグメント
- 下部にアノテーション一覧テーブル

セグメント:

- `face`
- 各 `FacePart` の `type`

初期状態:

- `face` が選択される

`face` 選択時:

- 顔全体画像を表示
- アノテーション一覧は非表示

部位選択時:

- 対象部位画像を表示
- 画像上にアノテーションを表示
- アノテーション一覧を表示

右上の編集ボタン:

- 現在状態に応じて画像変更を行う
- `face` のとき:
  - 顔全体画像を更新
- 部位選択時:
  - 対象部位画像が未設定なら追加
  - 設定済みなら更新

アノテーション一覧:

- 部位選択時のみ表示
- 各行に以下を表示
  - 注釈番号
  - 紐づいたカラーパレット画像
  - 紐づいたカラーパレット注釈タイトル

アノテーション一覧の操作:

- 行タップでアノテーション詳細画面へ遷移
- フッターの追加ボタンでアノテーション追加
- スワイプ削除でアノテーション削除

画像上の操作:

- 部位画像上のアノテーションはドラッグ移動できる
- ドラッグ終了位置を画像比率で保存する

## 6.5 アノテーション詳細画面

画面:

- `AnnotationDetailViewController`

表示内容:

- 選択中カラーの見出し
- 注釈タイトル表示
- 選択中カラーパレット名
- カラーパレット画像
- カラーパレット変更ボタン
- コメント入力欄

カラーパレット表示:

- 現在選択されている `selectedColorPalletID` の画像と注釈を表示
- 選択中の `selectedColorPalletAnnotationID` はハイライトする

操作:

- `カラーパレット変更` ボタンでカラーパレット選択モードの一覧へ遷移
- カラーパレット画像上の注釈をタップして使用色を選択
- コメントを編集して保存

カラーパレット変更モード:

- `MakeupLogListViewController(mode: .selectColorPallet)` を使用
- カラーパレットセクションのみを一覧表示
- 選択後に元画面へ戻る

キーボード対応:

- コメント入力中はスクロール領域の下余白をキーボード高に応じて調整する

## 6.6 カラーパレット編集画面

画面:

- `ColorPalletViewController`

表示内容:

- タイトル入力欄
- パレット画像
- 画像変更ボタン
- アノテーション追加ボタン
- 注釈一覧テーブル

操作:

- タイトル編集
- 画像差し替え
- アノテーション追加
- アノテーション位置ドラッグ移動
- 各アノテーションのタイトル編集
- 右上保存ボタンで更新保存

アノテーション一覧:

- 各行に注釈番号と説明入力欄を表示する
- 説明入力終了時に個別更新する

保存条件:

- タイトル必須

## 7. データ仕様

データ構造の詳細は `docs/agent-notes/DATA_MODEL.md` を参照します。

概要:

- `MakeupLog`
  - メイク記録のルート
- `FacePart`
  - 顔パーツ単位の子要素
- `FaceAnnotationObject`
  - 顔パーツ上の注釈
- `ColorPallet`
  - カラーパレット
- `ColorPalletAnnotationObject`
  - パレット上の色注釈

参照関係:

- 顔アノテーションは、選択中のカラーパレット ID と色注釈 ID を保持する
- これにより「顔のどの場所にどの色を使ったか」を記録できる

## 8. 保存仕様

### 8.1 永続化対象

- Realm にメタデータを保存する
- 画像本体はアプリの Documents 配下に保存する

### 8.2 画像保存

保存場所の考え方:

- `makeuplog/<uuid>.png`
- `facepart/<uuid>.png`
- `colorpallet/<uuid>.png`

画像圧縮仕様:

- PNG データサイズが 1MB を超える場合は JPEG 品質 0.5 で保存
- それ以下は PNG のまま保存

### 8.3 更新通知

保存や更新後は以下の通知を発行する。

- `Notification.Name.didLogUpdate`
- `Notification.Name.didColorPalletUpdate`

## 9. 初期データ仕様

メイクログ新規作成時:

- `eye`
- `lip`
- `cheek`

の 3 部位を初期 `FacePart` として自動生成する。

DEBUG ビルド時:

- インメモリリポジトリにサンプルデータが用意される
- サンプル顔画像、サンプルアイライン画像、サンプルカラーパレット画像を表示可能

## 10. アーキテクチャ

全体構成:

- `Views/`
  - 画面ごとの `ViewController` と `ViewModel`
- `Domain/`
  - Realm モデルおよび注釈関連モデル
- `DataStore/`
  - Repository プロトコルと実装
- `Service/`
  - 写真選択 UI などの補助機能
- `Utility/`
  - ファイル保存などの共通処理
- `Extension/`
  - `UIImage` 拡張や `Notification.Name` 拡張

責務分担:

- `ViewController`
  - 画面表示と UI イベントの受け取り
- `ViewModel`
  - 状態管理と画面ロジック
- `Repository`
  - Realm とファイル保存の操作

## 11. テスト状況

実装済みテスト:

- `MakeupLogRepositoryInMemory` に対する単体テスト
  - 取得
  - パーツ更新
  - 注釈更新
  - 注釈追加
  - パーツ追加
  - ログ追加

未整備または限定的な点:

- UI テストは雛形のみ
- カラーパレット系の単体テストは見当たらない
- 実 Realm 実装のテストは見当たらない

## 12. 現行仕様上の補足・制約

この章は「コード上そう見える」という意味での補足です。

- 画面遷移はすべて `UINavigationController` ベース
- 複数シーンは無効
- 一覧のセルは独自デザインよりも標準セル中心
- メイクログ詳細の画像表示領域は固定サイズに近いレイアウト
- アノテーションは画像内の比率座標で管理されるため、表示サイズが変わっても相対位置は維持される
- カラーパレット新規作成時は画像なしでも登録可能
- 顔アノテーションはカラーパレットを参照しなくても存在できる
